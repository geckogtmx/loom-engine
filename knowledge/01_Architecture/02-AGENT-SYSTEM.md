**Last Aligned:** 2025-12-12

> **Status:** AUTHORITATIVE  
> **Scope:** Engine-Wide (Agent Definition & Governance)

> **Precedence:** Subordinate only to Governance, Cognitive Architecture, and LOOM Architecture

---

## 1. Purpose

_(2025-12-12)_

This document defines **what an Agent is and is not** inside **LOOM**.

It removes ambiguity between:

- collaboration vs authority
    
- intelligence vs governance
    
- initiative vs autonomy
    

Agents are **execution instruments**, not independent actors.

---

## 2. Canonical Definition of an Agent

_(2025-12-12)_

An **Agent** is a **role-bound cognitive executor** operating inside a **World**, under a **Pattern**, within **META-enforced constraints**.

An Agent:

- performs delegated tasks
    
- applies specialized reasoning
    
- follows a fixed Telos
    
- escalates uncertainty
    

An Agent is **not**:

- a decision authority
    
- a policy maker
    
- a self-directed learner
    
- an evolving entity
    
- a governance layer
    

---

## 3. Agent Scope & Existence Rules

_(2025-12-12)_

- Agents **cannot exist outside a World**
    
- Agents **cannot operate outside a Pattern**
    
- Agents **cannot persist outside a Session**
    
- Agents **cannot access other Worlds**
    
- Agents **cannot modify system architecture**
    

If no World or Pattern is active, **Agents do not act**.

---

## 4. Agent Telos (Identity Kernel)

_(2025-12-12)_

Each Agent has a **Telos**, stored in **L4**.

The Agent Telos defines:

- role
    
- tone
    
- initiative bounds
    
- behavioral constraints
    
- escalation rules
    

### Telos Rules

- Telos is immutable during execution
    
- Telos may only be changed by the Operator
    
- META enforces Telos integrity
    
- Agents cannot reinterpret their Telos
    

Violation of Telos triggers escalation.

---

## 5. Initiative Model

_(2025-12-12)_

Agents operate under **bounded initiative**.

### Allowed

- execute assigned Pattern steps
    
- propose options when requested
    
- ask for clarification
    
- flag ambiguity
    

### Forbidden

- initiating new objectives
    
- redefining scope
    
- continuing without direction
    
- escalating complexity autonomously
    

When uncertain, the Agent must **pause and escalate**.

---

## 6. Agent Lifecycle

_(2025-12-12)_

Agents follow a **non-persistent lifecycle**:

`Defined (L3 + L4) ↓ Loaded into World ↓ Activated inside Pattern ↓ Execute during Session (L1) ↓ Contribute to L2 summary ↓ Session ends → Agent instance dissolves`

No Agent instance persists across Sessions.

---

## 7. Agent Evolution (Strictly Controlled)

_(2025-12-12)_

Agents **do not evolve autonomously**.

All evolution is governed by the **Controlled Evolution Framework**.

### Authority Chain

`Operator ↓ META (authorizes evolution) ↓ Agent University (specifies) ↓ META (validates) ↓ A0 (deploys)`

Agents:

- cannot self-modify
    
- cannot propose their own evolution
    
- cannot inherit memory implicitly
    

---

## 8. Agent Versioning

_(2025-12-12)_

- Agent versions are immutable
    
- Versions are parallel, not hierarchical
    
- Older versions remain addressable
    
- No version overwrites another
    

“Upgrade” means **replacement**, not mutation.

---

## 9. A0 (Agent Zero) Relationship

_(2025-12-12)_

A0 is **not a peer Agent**.

A0:

- supervises Agent behavior
    
- enforces constraints
    
- deploys approved versions
    
- escalates violations
    

Agents **cannot supervise A0**, only META and Operator can.

---

## 10. Agent Collaboration Rules

_(2025-12-12)_

- One lead Agent per Pattern step
    
- No Agent debates authority
    
- Agents do not negotiate scope
    
- Agents do not override each other
    

Collaboration is **sequenced**, not emergent.

---

## 11. Drift Detection & Handling

_(2025-12-12)_

Drift indicators include:

- tone deviation
    
- initiative overreach
    
- role confusion
    
- unauthorized persistence
    

### Response

- immediate halt
    
- escalation to Operator
    
- META review if repeated
    

No silent correction is allowed.

---

## 12. Canonical Constraints

_(2025-12-12)_

> **In LOOM, Agents execute.  
> They do not decide.  
> They do not auto-evolve.
