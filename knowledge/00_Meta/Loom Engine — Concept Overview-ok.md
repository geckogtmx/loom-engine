## What LOOM Is

The **LOOM Engine** is an **Operator-first cognitive framework** for structuring long-term work.

It enables a human Operator to run multiple projects as **isolated Worlds**, using **Patterns** and **Agents** under strict governance, while preserving identity, continuity, and intent across time.

LOOM is not a project manager, not an autonomous system, and not a self-evolving intelligence.

It is a **governed interaction platform** that structures how humans and AI collaborate.

---

## The Loom Metaphor (Explanatory)

The loom metaphor is descriptive only. It does not introduce system primitives.

- **Threads**  
    Ideas, notes, tasks, research, constraints, and decisions introduced by the Operator.
    
- **The Loom**  
    The Engine itself: the structured environment that applies order, pacing, and containment.
    
- **Patterns**  
    Canonical execution structures that define _how work unfolds_ inside a World.
    
- **Fabric (Worlds)**  
    Coherent project environments formed through repeated, governed execution.
    
- **The Operator**  
    The human authority who defines intent, identity, and final decisions.
    

---

## Why LOOM Exists

Modern knowledge work fragments cognition:

- mixed projects
    
- inconsistent context
    
- long conversations that drift
    
- loss of continuity
    
- unstable AI behavior
    

LOOM exists to restore **structure across time**.

It replaces improvisation with:

- isolation
    
- governance
    
- repeatable workflows
    
- explicit authority
    

---

## Operator-First Principle

**The Operator is the sole source of authority in LOOM.**

All other components exist to support, constrain, or execute the Operator’s intent.

There is no hidden agency, autonomous optimization, or self-directed evolution.

---

## Worlds: The Core Unit

A **World** is an isolated project environment with its own:

- purpose
    
- identity (Telos)
    
- rules and constraints
    
- Patterns
    
- Agents
    
- episodic continuity
    

Worlds do not share memory, authority, or execution context.

They are **stateful**, not autonomous.

---

## Patterns (Canonical)

A **Pattern** is a predefined execution structure.

Patterns:

- define workflow sequence
    
- constrain reasoning mode
    
- control pacing and output
    
- scaffold cognition
    

Patterns do not decide, adapt, or store memory.

They are activated by the ENGINE and executed within Sessions.

---

## Agents (Role-Bound Executors)

**Agents** are role-defined cognitive executors.

They:

- operate only within a World
    
- execute only within an active Session
    
- follow the active Pattern
    
- have no autonomous persistence
    
- do not define policy or goals
    

Agents exist to reduce Operator load, not replace judgment.

---

## Agent University (Regulatory System)

The **Agent University** is not an Agent.

It is a regulatory system responsible for:

- creating Agent definitions
    
- calibrating scope and tone
    
- proposing revisions
    
- managing versioning
    

The University does not enforce rules.

All changes require META authorization and are enforced by A0.

---

## A0 (Agent Zero)

**A0** is a special-purpose enforcement Agent.

A0:

- has no personality
    
- has no creativity
    
- does not interpret intent
    
- enforces META-authorized constraints at runtime
    

A0 never defines rules.

---

## META (Governance Layer)

**META** is the constitutional layer of LOOM.

META:

- defines what may exist
    
- authorizes change
    
- enforces constraints indirectly
    

META does not execute tasks and is not an Agent.

---

## ENGINE (Execution Plane)

The **ENGINE** is the global control plane.

It:

- initializes Sessions
    
- activates Worlds
    
- routes Patterns
    
- deploys Agents
    

ENGINE never overrides META or Telos.

---

## Memory Model (Layered)

LOOM uses a strict memory hierarchy:

- **L4 — Telos**: immutable identity and constraints
    
- **L3 — Knowledge**: structural definitions
    
- **L2 — Episodic**: condensed continuity
    
- **L1 — Active**: temporary session memory
    

Higher layers override lower layers.

---

## Operator Telos (Alignment Kernel)

The **Operator Telos** is the immutable identity layer.

It defines:

- values
    
- goals
    
- constraints
    
- working style
    

All Worlds, Patterns, and Agents operate within this boundary.

---

## Operator Growth Kernel (OGK)

The **OGK** is an observational analysis module.

It:

- analyzes pattern-level signals
    
- models cognitive rhythms
    
- produces insights only
    

OGK has no authority and modifies nothing.

---

## Sessions

A **Session** is a temporary execution envelope.

Sessions:

- initialize L1
    
- run a Pattern
    
- produce output
    
- summarize to L2
    
- terminate
    

Nothing persists without explicit summary.

---

## Feedback Loop (Procedural)

LOOM evolves only through a governed feedback process:

1. Operator reflection
    
2. META authorization
    
3. University proposal
    
4. A0 enforcement
    

There is no autonomous learning.

---

## Philosophy

LOOM is built on a single constraint:

> **Clarity across time requires structure.**

Creativity is supported by:

- isolation
    
- governance
    
- repeatability
    
- explicit authority
    

---

## Closing

LOOM is not an intelligence.

It is a **framework for thinking clearly with AI across time**, without surrendering control, identity, or intent.

This document explains _how it feels_ to use LOOM.

The **Glossary defines what it is**.
